# 反无技战法生成模块 - 设计文档

## 概述

反无技战法生成模块提供从基础模型到场景化可用模型的智能化生产与演进生成能力，构建以智能体为驱动、以人机协作为交互模式、以模型进化为目标的闭环工作系统。本模块设计为独立可嵌入组件，可集成到外部指挥平台或分析系统中。

### 1.2 接入层 - 统一工作空间

统一工作空间采用三栏布局，包含模型组件面板、画布区域和问答面板，实时同步：

```
┌──────────────────────────────────────────────────────────────────────────────┐
│ TopBar (48px)                                         任务ID │ 保存 │ 用户  │
├──────────┬─────────────────┬───────────────────────┬────────────────────────┤
│ SideNav  │ 模型组件面板     │    模型画布 (中央)      │   问答面板 (380px)     │
│ (60px)   │ (240px)         │   - 拖拽节点编排        │   - 自然语言交互       │
│ 图标模式  │ - 分类树形列表   │   - 模型依赖连线        │   - 状态查询           │
│          │ - 基础模型       │   - 实时状态标记        │   - 参数调整           │
│          │ - 场景模型       │   - 箭头类型标识        │   - 确认门禁           │
│          │ - 组合模型       │                      │   (SSE/WS 流式)        │
│          │ - 操作节点       │   (AntV X6 画布)      │                       │
├──────────┴─────────────────┴───────────────────────┴────────────────────────┤
│ 状态栏 (32px)                               阶段指示器 │ 进度 │ 引擎状态     │
└──────────────────────────────────────────────────────────────────────────────┘
```

**核心交互模式**：

| 区域 | 功能 | 交互方式 |
|------|------|---------|
| 模型组件面板 | 分类展示可用模型和操作节点 | 拖拽到画布创建节点 |
| 模型画布 | 可视化编排模型流程 | 节点拖拽、连线编辑 |
| 问答面板 | 需求描述、状态查询、参数调整 | 自然语言对话 |
| 协同模式 | 三向联动 | 组件面板↔画布↔问答实时同步 |

**模型组件面板分类**：
- **基础模型** — 通用预训练模型（目标检测、特征提取、文本理解等）
- **场景模型** — 场景微调模型（同行人识别、通信关联、整机采购等）
- **组合模型** — 碰撞产物模型（综合研判等）
- **操作节点** — 流程操作（模型优化、模型碰撞、验证评估）

**画布布局 — 决策树流程结构**：

画布采用自上而下的决策树布局，不体现内部架构分层。模型为**圆形节点**（直径60px），直接**嵌入**在优化框/碰撞框内部，在框内按**2列网格**排列。框与框之间用**直线**连接。框架**尺寸根据模型数量动态计算**（2个模型=1行≈140px高，4个模型=2行≈220px高）。

```
                    ┌─── 优化框 ────────┐        ┌─── 优化框 ────────┐
                    │ ●目标检测 ●文本理解  │        │ ●文本理解 ●特征提取  │
                    │  → 输出优化模型 A    │        │  → 输出优化模型 B    │
                    └─────────┬──────────┘        └─────────┬──────────┘
                              │                             │
                              │ (直线)                       │ (直线)
                              └────────────┬────────────────┘
                                           ▼
                                 ┌─── 碰撞框 ───────────┐
                                 │ ●优化模型A  ●优化模型B │  ← 优化产物仅在框内
                                 │  → 输出碰撞新模型      │
                                 └─────────┬─────────────┘
                                           │ (直线)
                                           ▼
                                       ●碰撞新模型
```

**节点类型**：
- **模型节点**（圆形，直径60px）— ● 蓝色实线圆=原始模型，● 橙色虚线圆=生成模型，标签在圆心
- **框架节点**（虚线大容器，带标题栏）— 紫色=优化框，红色=碰撞框
  - 尺寸按内部模型数量动态计算：宽=22×2 + 2×60 + 20 = 184px，高=40 + rows×74 + 36
  - 2列网格排列，奇数行自动居中

**连线样式**：
- **直线连接**，仅连接框架到产物/产物到框架
- 标签标注操作类型（生成、碰撞等）

**画布交互**：
- **滚轮缩放** — 自由缩放 0.3x~2.5x
- **拖拽平移** — 左键拖拽空白区域平移
- **点击框架节点** → 跳转对应工作台，操作记录与画布同源

### 1.3 接入层 - 人机协同渐进式工作流

每一步都需要用户确认，系统是"建议者"，分析师是"决策者"：

```
分析师启动分析任务
       │
       ▼
┌──────────────────────┐
│  ① 模型分配           │ ← 检索已有模型，匹配当前场景
│  检索结果:             │
│  - 匹配模型 (可用)     │
│  - 不匹配 / 无模型     │
└──────────┬───────────┘
           │ 不满足
           ▼
┌──────────────────────┐
│  ② 自动推荐优化方案    │ ← 系统推荐，分析师确认
│  [确认优化] [拒绝]    │
└──────────┬───────────┘
           │ 确认
           ▼
┌──────────────────────┐
│  ③ 模型优化           │ ← 跳转独立训练界面
│  优化完成 → 验证结果   │
│  [满足→碰撞] [否→再优化]│
└──────────┬───────────┘
           │ 满足/确认
           ▼
┌──────────────────────┐
│  ④ 模型碰撞           │
│  多模型联合推理        │
│  → 碰撞规则沉淀        │
│  → 新模型资产          │
└──────────────────────┘
```

**关键特点**：
- 不一定走完全流程：已有模型满足需求可直接跳到碰撞
- 回退机制：优化不满足可继续迭代
- 用户确认门禁分布：模型分配后、优化推荐时、优化完成后、碰撞完成后

## 2. 智能体协同引擎

### 2.1 智能体组成与职责

| 智能体 | 核心职责 | 触发时机 |
|--------|---------|---------|
| 模型优化统筹智能体 | 任务拆分、工作流编排、状态管理、各智能体调度 | 任务创建时启动，贯穿全流程 |
| 模型检索问答智能体 | 语义搜索模型资产、匹配度评估、推荐理由生成、不满足分析 | 模型分配阶段 |
| 数据集构建智能体 | 数据采集管道、数据标注、数据增强、版本管理 | 需要训练/优化时 |
| 模型优化智能体集 | 基础模型加载、超参搜索、分布式训练、模型转换 | 进入优化阶段 |
| 验证智能体 | 交叉验证、对抗验证、场景回放、边界测试 | 优化完成后 |
| 评估体系智能体 | 多维度评估、对比分析、报告自动生成 | 验证通过后 |
| 模型碰撞统筹智能体 | 推理调度、结果融合、规则提取沉淀 | 用户确认进入碰撞 |

### 2.2 智能体间通信协议

统一消息格式：

```json
{
  "task_id": "TASK-2024-001",
  "agent_from": "orchestrator",
  "agent_to": "retrieval_agent",
  "action": "search_model",
  "payload": {
    "scene": "同行人识别",
    "scenario": "城市反无人机",
    "constraints": {"latency": "<100ms"}
  },
  "context_id": "CTX-001"
}
```

### 2.3 混合部署策略

| 组件 | 本地 (边缘) | 云端 |
|------|-----------|------|
| 画布前端 | ✅ | - |
| 问答面板 | ✅ | - |
| 模型检索问答智能体 | ✅ 轻量检索 | ✅ 全量检索 |
| 模型优化统筹智能体 | ✅ 编排调度 | - |
| 数据集构建智能体 | ✅ 小样本 | ✅ 大规模数据 |
| 模型优化智能体集 | - | ✅ GPU 训练 |
| 模型碰撞统筹智能体 | ✅ 推理 | ✅ 大规模碰撞 |
| 模型资产存储 | ✅ 缓存热模型 | ✅ 全量存储 |

## 3. 模型资产层

### 3.1 资产组成

| 仓库 | 内容 | 说明 |
|------|------|------|
| 基础模型库 | YOLOv8, ResNet, BERT, TemporalNet 等 | 通用预训练模型 |
| 场景模型库 | 同行人识别v1/v2, 涉机言论检测v2, 整机采购识别v1 等 | 场景微调后模型 |
| 碰撞规则库 | 同行+通信→高危, 背包+遥控→准备, 多车驻留→集结 等 | 碰撞推理沉淀的规则 |
| 数据集仓库 | 原始数据集、标注数据集，支持数据版本管理 | 训练数据 |

### 3.2 三维分类体系

```
维度1 - 侦测阶段:
  行动策划类 → 人员集结类 → 装备筹措类 → 物流运输类 → 临期准备类

维度2 - 模型形态:
  基础模型 (通用) → 场景模型 (微调后) → 组合模型 (碰撞产物)

维度3 - 成熟度等级:
  L1-实验验证 → L2-场景验证 → L3-业务可用 → L4-生产就绪
```

### 3.3 碰撞规则沉淀机制

```
多模型碰撞推理
      ↓
提取碰撞发现 (如: "同行人出现 + 频繁通信 → 高危")
      ↓
结构化规则 (置信度/触发条件/建议行动)
      ↓
规则入库 → 关联场景标签 → 新规则可用于后续碰撞
      ↓
规则回馈 → 优化基础模型 (闭环)
```

## 5. 异常处理与容错

| 场景 | 处理方式 |
|------|---------|
| 模型检索无结果 | 自动触发"冷启动"流程，推荐从基础模型开始训练，告知分析师预期耗时 |
| 训练失败 | 保存断点，记录失败原因（数据问题/参数问题/资源问题），提供重试/调整选项 |
| 碰撞结果异常 | 标记碰撞结果低置信度，询问分析师是否接受或调整模型组合 |
| 网络断连（混合部署） | 本地缓存任务状态，恢复后自动同步；云端训练任务继续运行不中断 |
| 用户中途关闭 | 保存任务快照，支持恢复继续编排 |

## 6. 可嵌入性设计

### 6.1 集成方式

| 方式 | 说明 |
|------|------|
| Python SDK | `pip install mechgen-engine`，提供核心引擎 API |
| Docker 容器 | 一键部署引擎服务 |
| REST API / gRPC | 外部系统通过 HTTP/gRPC 调用核心能力 |
| Web Component | 画布和训练界面可作为组件嵌入其他系统 |

### 6.2 API 接口示例

```
POST /api/v1/tasks          — 创建分析任务
GET  /api/v1/tasks/{id}     — 查询任务状态
GET  /api/v1/models         — 检索模型资产
POST /api/v1/collision/run  — 执行模型碰撞
```

## 7. 场景化模型清单
